# 电商客户流失预测

## 项目简介

在竞争激烈的电商行业，客户流失是企业面临的核心挑战之一。获取新客的成本远高于留存老客。本项目的目标是通过机器学习模型，基于客户的历史行为数据，**提前识别有流失风险的客户**，并找出导致流失的关键因素，从而帮助业务团队制定精准的挽留策略，降低流失率，提升客户生命周期价值。

## 数据集说明

- **样本量**：15,000 条客户记录
- **特征数**：11 个特征（含数值型和类别型）
- **目标变量**：`Is_Churn`（1 = 流失，0 = 未流失）
- **流失率**：约 9.55%，存在较明显的**类别不平衡**问题

主要特征包括：年龄、订阅月数、合同类型、每月登录次数、距上次购买天数、App使用时长、月消费金额、折扣使用比例、客服呼叫次数、满意度评分。

## 项目流程

1. **探索性数据分析 (EDA)**
   - 流失分布可视化（饼图）
   - 数值特征分布直方图
   - 类别特征分析
   - 相关性热力图，发现 `Customer_Support_Calls`、`Subscription_Duration_Months` 等与流失高度相关

2. **数据预处理**
   - 将 `Contract_Type` 进行独热编码
   - 对数值特征进行标准化（`StandardScaler`）

3. **处理不平衡**
   - 使用 **SMOTE**（合成少数类过采样技术）生成新的流失样本，平衡训练集

4. **建模与评估**
   - 模型：逻辑回归、随机森林
   - 评估指标：准确率、召回率、F1 分数、ROC‑AUC
   - 交叉验证确保模型稳定性

5. **模型解释（SHAP）**
   - 使用 SHAP 值解释随机森林模型的预测结果
   - 识别对流失预测贡献最大的特征

## 主要结果

| 模型 | ROC‑AUC | 召回率（流失类） |
|------|---------|----------------|
| 逻辑回归 | 0.95 | 0.88 |
| 随机森林 | **0.98** | **0.89** |

- **最佳模型**：随机森林
- **最重要的流失特征（TOP 3）**：
  1. `Customer_Support_Calls`（客服呼叫次数）
  2. `Subscription_Duration_Months`（订阅月数）
  3. `Last_Purchase_Days_Ago`（距上次购买天数）

> SHAP 分析显示，客服呼叫次数 ≥ 4 次、订阅月数 ≤ 3 个月、距上次购买 > 15 天的客户流失概率显著升高。

## 业务建议

1. **对高客服呼叫客户主动关怀**：当客户在短时间内呼叫客服 ≥ 3 次时，自动触发客户关怀流程（如发放优惠券、专属客服回访）。
2. **优化新客初期体验**：针对订阅 ≤ 3 个月的新客户，推送新手教程、定期推送个性化推荐，提升初期粘性。
3. **激活沉默客户**：对距上次购买超过 15 天的客户发送限时折扣推送，并配合“积分即将过期”等提示，刺激再次消费。
4. **建立流失预警机制**：将模型评分前 15% 的高风险客户名单每周同步给运营团队，进行定向干预。
