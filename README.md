# 电商客户流失预测

## 项目简介

在竞争激烈的电商行业，客户流失是企业面临的核心挑战之一。获取新客的成本远高于留存老客。本项目的目标是通过机器学习模型，基于客户的历史行为数据，**提前识别有流失风险的客户**，并找出导致流失的关键因素，从而帮助业务团队制定精准的挽留策略，降低流失率，提升客户生命周期价值。

## 数据集说明

- **样本量**：15,000 条客户记录
- **特征数**：11 个特征（含数值型和类别型）
- **目标变量**：`Is_Churn`（1 = 流失，0 = 未流失）
- **流失率**：约 9.55%，存在较明显的**类别不平衡**问题

主要特征包括：年龄、订阅月数、合同类型、每月登录次数、距上次购买天数、App使用时长、月消费金额、折扣使用比例、客服呼叫次数、满意度评分。

## 项目流程

1. **探索性数据分析 (EDA)**
   - 流失分布可视化
   - 数值特征分布直方图
   - 类别特征分析
   - 相关性热力图，发现 `Customer_Support_Calls`、`Subscription_Duration_Months` 等与流失高度相关

2. **数据预处理**
   - 将 `Contract_Type` 进行独热编码
   - 对数值特征进行标准化（`StandardScaler`）

3. **处理不平衡**
   - 先拆分训练集/测试集，再对训练集使用 **SMOTE** 过采样，避免数据泄露

4. **基线建模**
   - 逻辑回归（AUC 0.92）
   - 随机森林（AUC 0.91）
   - 对比后选用随机森林继续优化

5. **模型优化**
   - **5折交叉验证**：评估模型稳定性（f1: 0.9087 ± 0.0096）
   - **网格搜索（GridSearchCV）**：108 组参数组合，以 ROC-AUC 为优化目标
   - 最佳参数：`max_depth=None, n_estimators=300, min_samples_split=2, min_samples_leaf=1`

6. **RFM 客户分层**
   - 基于最近消费（R）、登录频率（F）、消费金额（M）将客户分为 4 层：
     - 高价值客户 / 一般价值客户 / 待唤醒客户 / 低价值客户

7. **干预优先级矩阵**
   - 将 RFM 分层与模型预测的流失概率结合，构建 4×3 干预矩阵
   - 制定分层干预策略（专属客服回访、折扣券推送、VIP 福利等）
   - **ROI 估算**：每投入 1 元挽回成本，预期回报数元

## 主要结果

| 模型 | ROC-AUC | 准确率 |
|------|---------|--------|
| 逻辑回归 | 0.92 | 0.88 |
| 随机森林（基线） | 0.91 | 0.88 |
| 随机森林（调优后） | 0.90 | 0.89 |

> 调优后准确率微升，但核心价值在于 RFM 分层与干预策略将模型输出转化为可执行的业务动作。

### 最重要的流失特征（TOP 3）

1. `Customer_Support_Calls`（客服呼叫次数）
2. `Subscription_Duration_Months`（订阅月数）
3. `Last_Purchase_Days_Ago`（距上次购买天数）

> 客服呼叫次数 ≥ 4 次的客户流失率超 50%，订阅 ≤ 3 个月且距上次购买 > 15 天构成高危画像。

## RFM 客户分层结果

| 分层 | 占比 | 特征 | 策略 |
|------|------|------|------|
| 高价值客户 | ~5% | R高 + F高 + M高 | 保持满意度，预防降级 |
| 一般价值客户 | ~14% | 各方面中等偏上 | 个性化推荐提升粘性 |
| 待唤醒客户 | ~73% | 某方面突出但整体不足 | 选择性召回 |
| 低价值客户 | ~7% | R低 + F低 + M低 | 不干预，接受流失 |

## 业务建议

1. **优先干预**：高价值 + 高风险客户 → 专属客服回访 + 大额优惠券
2. **预防维护**：高价值 + 中风险客户 → VIP 福利推送，防止降级
3. **激活沉默客户**：对距上次购买超过 15 天的客户发送限时折扣推送
4. **建立流失预警机制**：将高风险客户名单每周同步给运营团队，进行定向干预
